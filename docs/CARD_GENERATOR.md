# 卡牌生成器使用说明

## 概述

卡牌生成器可以批量生成700张卡牌，满足以下规格：

### 元素分布
- **五行元素**（金、木、水、火、土）：每个元素100张，共500张
- **光暗元素**（光、暗）：每个元素50张（红色45张+彩色5张），共100张
- **特殊角色**：保留6张初始设计角色
- **总计**：约700张卡牌

### 品质分布（每元素100张）
| 品质 | 数量 | 比例 | 说明 |
|------|------|------|------|
| 绿色 | 40张 | 40% | 基础卡牌 |
| 蓝色 | 30张 | 30% | 优秀卡牌，有被动技能 |
| 紫色 | 15张 | 15% | 史诗卡牌，有觉醒形态 |
| 金色 | 10张 | 10% | 传说卡牌 |
| 红色 | 5张 | 5% | 神话卡牌 |
| 彩色 | - | - | 光暗专属，各5张 |

**光暗元素特殊规则**：最低品质为红色（45张红色+5张彩色）

### 流派分布
包含22种战斗流派，每种流派卡牌属性倾向不同：

**输出流派**：暴击流、连击流、毒素流、灼烧流、群攻流、爆发流、斩杀流、吸血流
**控制流派**：眩晕流、冰冻流、沉默流、嘲讽流
**防御流派**：闪避流、护盾流、反弹流、回复流
**辅助流派**：治疗流、增益流、减益流、复活流
**特殊流派**：献祭流、召唤流、复制流

## 使用方法

### 1. 生成卡牌数据

在Cocos Creator项目中调用：

```typescript
import { generateAllCards } from './assets/scripts/generator/CardGenerator';

// 生成所有卡牌
const allCards = generateAllCards();
console.log(`生成了 ${allCards.length} 张卡牌`);

// 按元素筛选
const metalCards = allCards.filter(card => card.element === ElementType.METAL);

// 按品质筛选
const redCards = allCards.filter(card => card.rarity === Rarity.RED);

// 按流派筛选（需要额外计算）
```

### 2. 生成统计信息

```typescript
import { generateStatistics } from './assets/scripts/generator/CardGenerator';

const stats = generateStatistics();
console.log(stats);
// {
//   totalCards: 700,
//   byElement: { metal: 100, wood: 100, ... },
//   byRarity: { green: 200, blue: 150, ... }
// }
```

### 3. 动态加载卡牌

由于卡牌数量庞大（700张），建议采用**动态生成**而非**静态存储**：

```typescript
// 不要一次性加载所有卡牌
// 而是按需生成或分页加载

// 示例：只加载玩家已拥有的卡牌
const ownedCardIds = ['metal_1', 'fire_50', 'light_5'];
const allCards = generateAllCards();
const ownedCards = allCards.filter(card => ownedCardIds.includes(card.id));
```

## 卡牌ID命名规则

```
{元素}_{序号}

例如：
- metal_1 ~ metal_100：金元素100张卡牌
- wood_1 ~ wood_100：木元素100张卡牌
- light_1 ~ light_45：光元素红色卡牌
- light_rainbow_1 ~ light_rainbow_5：光元素彩色卡牌
```

## 属性生成规则

### 基础属性公式
```
基础值 = 基础数值 × 流派系数 × 品质系数

基础数值：
- HP: 2500-3000
- ATK: 280-340
- DEF: 160-200
- SPD: 95-115

品质系数：
- 绿色：1.0x
- 蓝色：1.2x
- 紫色：1.5x
- 金色：1.8x
- 红色：2.2x
- 彩色：2.8x
```

### 流派属性倾向
| 流派 | 属性倾向 |
|------|----------|
| 暴击流 | 暴击率+20%，暴击伤害+180% |
| 连击流 | 速度+20% |
| 控制流 | 效果命中+20% |
| 闪避流 | 速度+15%，闪避率+25% |
| 防御流 | 防御+30%，生命+30% |
| 治疗流 | 效果抵抗+15% |
| 献祭流 | 攻击力+30% |

## 技能生成规则

每个流派有预设的技能模板库：
- **普通攻击**：2-3个模板，随机选择
- **特殊技能**：2-3个模板，消耗能量30-50
- **被动技能**：2-3个模板，蓝色及以上品质才有

技能效果根据流派特性自动生成，例如：
- 毒素流：附加中毒效果
- 眩晕流：概率眩晕目标
- 吸血流：造成伤害恢复生命

## 注意事项

1. **不要提交生成的700张卡牌数据到Git**（文件太大）
2. **运行时动态生成**，或
3. **使用服务器存储**卡牌数据库
4. **初始只加载玩家已拥有的卡牌**

## 扩展建议

如需添加新流派：
1. 在 `CardArchetype` 枚举中添加新流派
2. 在 `ARCHETYPE_TRAITS` 中定义流派特性
3. 在 `SKILL_TEMPLATES` 中添加技能模板
4. 在 `archetypeStats` 中定义属性倾向

如需调整品质分布：
修改 `GENERATION_CONFIG.rarityDistribution` 中的比例

## 性能优化

对于700张卡牌的性能优化建议：

1. **延迟加载**：只在需要时生成卡牌数据
2. **缓存机制**：缓存已生成的卡牌，避免重复计算
3. **分页显示**：图鉴界面分页加载，每页20-50张
4. **数据压缩**：传输时使用压缩格式
5. **服务器存储**：将生成逻辑放在服务器，客户端只接收需要的卡牌数据
